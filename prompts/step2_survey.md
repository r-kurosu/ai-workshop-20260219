# Step 2: 施策シミュレーション

生成済みペルソナに施策3パターンを提示し、反応を取得するスクリプトを作って実行してください。

## 入力

`output/personas.json`

## サービス

AI家計簿アプリ「カケイロ」（レシート撮影→自動入力→AI分析→節約提案→月次レポート）

## 施策3パターン

- **A（フリーミアム）**：「ずっと無料。広告が表示されます。レシート撮影と基本的な収支管理が使えます。」
- **B（スタンダード）**：「月額980円。広告なし。AIが支出パターンを分析し、あなた専用の節約プランを毎週提案します。」
- **C（ファミリー）**：「月額1,980円。家族全員のアカウントを連携。世帯全体の家計をAIが最適化。教育費・老後資金のシミュレーション付き。」

## 質問（各施策に同じ3問 + 施策横断で1問）

施策別：
- Q1: 試してみたいか？（1〜5）
- Q2: その理由（2-3文）
- Q3: この価格は妥当か？（1〜5）

施策横断（全施策提示後に1回）：
- Q4: 3つのプランの中でどれを選ぶか？（A/B/Cで回答）＋選んだ理由（1文）

## LLMへの回答指示（重要）

ペルソナになりきらせる際に、以下を明示すること：
- 「このペルソナの家計状況、過去のアプリ体験、広告への態度を踏まえて回答してください」
- 「中間的な回答（3）を安易に選ばず、このペルソナならではの明確な態度を示してください」
- 「過去にアプリで失敗した経験がある人は、その経験が回答に影響するはずです」

## 要件

- Azure OpenAI API (gpt-4o-mini) を使用
- 50人 × 3施策 = 150回呼び出し（Q1〜Q3）。Q4は別途50回呼び出し
- 進捗をコンソール表示

## 出力

`output/responses.json`：

```json
{"persona_id": "P001", "plan": "A", "q1": 4, "q2": "無料なら気軽に試せる。", "q3": 5}
```

`output/choices.json`：

```json
{"persona_id": "P001", "choice": "B", "reason": "広告なしで節約提案があるのが決め手。"}
```
